二、调整各报生成的的流程
日报：
    允许提供全量 plan.md 内容，默认以当前 plan.md 为准（直接变更）进行对比
周报：
    周报周期：周报周期为上周五到本周四，上周周报优先以线上周报为准
    周报内容：
        基于周期的 plan.md 内容进行对比，提取新增内容，生成周报草稿；
        原始进度以上周周报最新进度为准（新增项为0%）， 按设计-开发-测试-上线的 20%-50%-80%-100% 进行进度更新；
月报：
    月报内容：
        基于月度的 plan.md 内容进行对比，提取新增内容，生成月报草稿；
        原始进度以上月月报最新进度为准（新增项为0%）， 按设计-开发-测试-上线的 20%-50%-80%-100% 进行进度更新；
        其中月度 OKR 默认复制上月月报 OKR 内容，并根据本月日报进度更新进度； 各季度第一月或OKR 不存在时，需由用户提供 OKR 内容或链接等信息

三、推送流程
在与用户完成日/周/月报确认后，进行后续推送操作
各场景都包含：
    推送 git：将本次变更推送到 git 仓库，并更新 commit 信息

周报同步：
    推送乐享： 按线上周报目录层次，使用乐享 mcp 将上周周报复制到对应目录下，并更新内容； 注意不支持复制到的情况下，则新增文档并更新标题即可
        周报目录下层次： 预期层次为 02.个人周报/ {年份，如20026} - 监控组个人周报/ {周期，如2026.04.10-2026.04.16}【监控组个人周报】， 对应层级不存在时自行建立
月报同步
    推送乐享： 按月报工作复盘层次，进行月报更新
        月报目录下层次： 预期层次为 03.工作复盘/ {年份，如20026}工作复盘（监控组）/ 姚清居 -{周期，如2026.03} 工作复盘 (监控组)， 对应层级不存在时自行建立

补充说明： 线上周报根目录、月报根目录地址和 mcp 配置信息统一走配置； 未提供找用户确认
