# Report Templates

Reference: `config/system.json` -> `paths.reportTemplatePath`

If `paths.reportTemplatePath` is missing, ask the user to provide the template path first.

## Daily Report

成稿规则（事项名标题、禁止 `### 工作事项`、工时换算、1.5d 上限与均分、luckin/daily 合并、脚本开关）以 **SKILL「日报格式控制」** 为准；本处仅保留最小骨架示例。

```md
## 日报｜「日期，如 2026-04-17」

### 「具体事项名，如 答疑」
- 标签： 「如 国内、国际化、排障、架构、AI探索、优化、答疑、其它」
- 工时： 「如 3h、1d」（无则省略本行；冒号后空一格）
- 完成细项
    - 「细项说明 1」
    - 「细项说明 2」
- 文档
    - 「设计文档或上线单链接」

### 其它
- 「其它事项概述」
```

## Weekly Report

**本地 `record` 周报以分层 `### 具体事项名` 为标题**（可读性优先）。`- 进度：` 与上周同事项末值对比（乐享同一事项优先；本地验证时 `- 对照：` 摘录上周 md 原文）；**起止百分数相同**时写单一 `n%`（如 `100%`），不写 `100%->100%`。`- 说明：` 体现**最新**进度与关键事项；**上线/发版**写清版本号或 Launch（有则写），不单列占位句。同步乐享时可改表格式。

`完成细项`：按**功能/事项**合并本周日报，**禁止**按自然日前缀；**优先合并描述**，必要时用**分类**（上线与版本 / 开发与验证 等）或**一层子级**，避免同级平铺错乱。

```md
## 周报｜「周期，如 2026.04.10-2026.04.16」

> 周期：**上周五–本周四**。乐享对照：[乐享页面](https://lexiangla.com/pages/cd393fac5ec84ff99c6c319fd86b72c0)。

### 对比上周
- 「乐享 / 本地 2026-W15.md；基线说明」

### 「具体事项，如 故障评分 hotfix」
- 进度：「有变化：80%->100%；无变化：100%」
- 对照：「上周同事项进度为「…」（摘自 `2026-W15.md`；与乐享不一致时请改正文）」（上周无该事项时可省略）
- 说明: 「最新进展与关键路径；上线写版本/Launch」
- 完成细项
    - 「上线与版本：…；…」（或分层：组标题 + 子项）
- 文档
    - 「链接」

### 其它
- 「本周期含日报 / 缺日报日期；LUCP 补录提示等」
```

## Monthly Report

月报内容须完整可展示。OKR 表同一 O 下多行 KR 时，O 列首行填完、后续行留空；上线表同一服务多版本时，服务名称每行直接填值即可，不依赖额外占位说明。

```md
## 一、OKR
| **Current.O（子目标）** | KR（关键工作项） | 进度 | **备注** |
|-------------------------|------------------|------|----------|
| O：「O 描述」 | 「KR 描述，多行用 <br>」 | 「进度，如 33%→66%」 | 「补充说明，多条用 <br>；未涉及略过」 |
| | 「同 O 下续 KR，O 列留空」 | | |
| O：「O 描述」 | 「KR 描述，多行用 <br>」 | 「进度，如 33%→66%」 | 「补充说明，多条用 <br>；未涉及略过」 |
| | 「同 O 下续 KR，O 列留空」 | | |

## 二、其他有意义的或重要工作项
- 「工作事项」

## 三、本月上线、发版本（二方包、launch等）
| 服务名称 | 版本 | 功能描述 | 线上变更 |
|----------|------|----------|----------|
| 「服务名称」 | 「版本号」 | 「功能概述」 | [Launch-xxx](https://lucp.lkcoffee.com/launch/Launch-xxx) |
| 「服务名称」 | 「版本号」 | 「功能概述」 | [Launch-xxx](https://lucp.lkcoffee.com/launch/Launch-xxx) |

## 四、本月事故、事件情况

## 五、本月主要完成的文档情况
- 「文档列表」

## 六、总结、经验、成长、收获等
- 「总结内容」
```

## Shoot-Weekly (Troubleshooting Iteration)

```md
### 「迭代方向，如触发能力、分析能力、生命周期管理、其它」
- 「包含整体进度说明、关键细节项」
    - 「按需求大小补充细节项」
    - 「补充设计文档、跟进文档及说明」
```

Extract from daily reports within the time range, group by iteration direction, and summarize progress plus supporting details.
