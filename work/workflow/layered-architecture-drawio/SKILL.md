# 分层式系统架构图（设计规划）

## 与样式规则的关系

- **本 SKILL**：**做什么叙事、怎么分层、先看什么文档**（规划体）。
- **样式专规**：本仓库根下 `.cursor/rules/layered-architecture-drawio.mdc`（**配色 L0～L4、线型、画布分栏、块/备注/底栏图说、v3 虚线外框**）。生成或改 `.drawio` 时**必须先读**该规则再落笔具体 `fillColor`/`strokeColor`/`strokeWidth`/几何。

## 何时使用

- 自上而下或左–中–右分栏的**系统架构总览**（非泳道、非步进时序）。
- 需与《总体技术方案》等**同一套说法**（单源、中部能力、宿主、外联、存量对比）。

## 设计规划

### 1. 与方案对齐（先于画图）

- 从方案中抽出并**固定**下列用语，图内**不得换一套话**：单源/收敛点、能力面所在栏、双宿主或等价入口、外联/观测、存量/过渡态（若方案有写）。
- 文内 Mermaid/章节标题与图要**一一可对照**，避免「文里中台、图里又写绿层」等漂移。

### 2. 层次抽象

- 用 **L0～L4 语义**（抽象→可执行→外联）给每个「主栏」**定阶**，再填子块；**具体色号与线粗**以规则文件为准，此处只定**栏的角色**（宿主衬底/资产与方法论/卡片与核心/外部等）。
- **主链**：先标「谁先进单源能力、再出到外联/观测」；**约束/资产/方法论** 与主链的区分用**线型**表达，见规则。
- **存量/非目标**：单独成带或块，**对比叙事**、不占主色阶主线。

### 3. 四类对象与顺序

- **主节点**：大分栏/泳道式分组（本 skill 的架构图**不含多泳道时序**，但可有**大矩形分栏**）。
- **子节点**：栏内业务块，**同级等高/等宽**由**最长文案**定基准（细则见规则）。
- **备注**：`note`，叠在**被说明子块或栏**的右上，不与大块**假横排**。
- **图形说明**：**页底独立带**（色阶+图例+读图句），不挤在图心。

**推荐落地顺序**：列对象类型 → 定分栏与主链 → 定子块文案与尺寸 → 叠备注 → 再摆底栏 → 再连线（含避让）。

### 4. 出图前检查

- 是否仍属「分层/分栏总览」；若已是**时序/泳道步进**则**不**用本套，换通用 drawio 或其它规范。
- 主链是否**一眼**从宿主汇到**中部能力**再到外联/观测。
- 底栏是否**单独一条**，且未用长 Note 替代。

## 执行要点

1. 打开并遵循 `.cursor/rules/layered-architecture-drawio.mdc` 中的**色、字、线、画布、v3 外框**。
2. 按上文**设计规划**定栏位与主链，再写 cell 与 edge。
3. 与《方案》有出入时，**以方案与产品表述为准**改图，而不是改口独自洽。

## 不适用范围

- **多泳道、交互时序、步进流** 类图：不由本 skill 与配套样式规则作为唯一依据。
